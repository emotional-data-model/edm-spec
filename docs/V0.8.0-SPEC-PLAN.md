# EDM v0.8.0 Specification Plan

## What changes

### 1. meta.profile — two-tier enum
Currently: hard enum (essential|extended|full)
v0.8.0: z.union([z.enum([...canonical...]),
  z.string()]).nullable()
Same pattern as arc_type in v0.7.0.
Non-canonical values trigger registry
resolution.

### 2. Section 3.7.6 — Partner Profiles
New section in PROFILES.md.
Partner profile is a named, versioned
schema declaration. Minimum 10 affective
fields from the 57-field affective set.
Defines: profile_id format, registry
resolution protocol, certification bar,
interoperability rules.

### 3. Schema files
New files using v0.8.0 naming:
- edm.v0.8.essential.schema.json
- edm.v0.8.extended.schema.json
- edm.v0.8.full.schema.json
meta.profile field updated in all three.

### 4. Partner profile examples
